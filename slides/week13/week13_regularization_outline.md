# 幕一：OLS 还不够——为什么模型需要“约束”？

## 本幕定位
这是整节课的 prologue + 第一幕。
这一幕要先用图把“为什么还要折腾线性模型”讲清楚，而不是立刻进入定义。

> 我们上节课讲了 Bias-Variance Trade-off，用汽车来比喻就是：
- Bias 高了开的不快；
- Variance 高了开的不稳；
- 我们的目标自然是又快又稳；
- 如何衡量？跑一个之前没见过的赛道（泛化能力）。

虽然，从我们上节课的例子中可以看到：和各种 ML 模型比起来，Linear Model 已经是老爷车了——它的 bias 很高；并且在“多重共线性”时，它也不稳。

那么今天，我们还是以 Linear Model 为例，聚焦：**怎么让模型更稳（Variance 更小）**。

## 本幕主图
**OLS 系数不稳定图**

建议把这张图作为本幕的主舞台，直接回答：
(注意，在代码中这里留一个空cell即可，我会引用学生上周的作业)
> 同一个建模任务里，只要样本略有变化，OLS 的系数会不会明显波动？

这张图要教会学生的核心判断：
- OLS 不是不能拟合；
- 但在相关特征存在时，系数可能不稳定；
- 问题不只是“误差高不高”，还包括“模型结论会不会乱跳”。

## 围绕主图的课堂问题
- 如果你要把一个回归模型交给业务团队长期使用，你更担心“训练误差不够低”，还是“模型太不稳定”？
- 如果换一批样本，系数方向或大小明显变化，这对解释和决策意味着什么？
- 当几个变量高度相关时，OLS 给出的“谁更重要”还能轻易相信吗？

## 本幕 takeaway 应落到
- 我们今天要解决的问题不是“OLS 错了”，而是“OLS 在某些场景下不够稳”；
- 一旦模型自由度太高，就需要某种约束来换取稳定性与泛化能力；
- 这就是正则化要登场的原因。

## 教师备注
- 这一幕不要过度证明，只要把 stakes 立住；
- 如果学生对“稳定性”不敏感，可以用“同一套业务规则，换一批样本就给出另一套结论”来解释风险；
- 不建议把抽象的 Bias-Variance 曲线当作这一幕主图，它更适合回顾，不适合作为本幕证据。

# 幕二：统一框架——`loss + penalty`

## 本幕定位
这一幕是全节课的认知支点。

继上节课“Loss（RMSE vs. MAE）塑造模型”之后，继续看“Loss with Penalty 如何塑造模型”。
这一幕不追求图多，而要用图把一个统一语言立住。

## 核心冲突
> 我们为什么要故意让模型别那么自由？多出来的 penalty 到底是在惩罚什么？

## 本幕主图
**`loss + penalty` 的结构图**

这张图不是为了展示复杂数学，而是要把一个判断钉住：
- `loss` 负责拟合数据；
- `penalty` 负责约束参数规模；
- 正则化不是换模型世界，而是在原目标函数里加入复杂度成本。

这张图要教会学生：
> 后面所有方法的共同语言，都是“拟合误差 + 复杂度约束”。

## 围绕图的课堂问题
- 如果惩罚项很小，模型会更像谁？
- 如果惩罚项越来越大，系数、拟合能力、稳定性可能分别发生什么变化？
- 你愿不愿意为了更稳，牺牲一点训练集上的贴合度？

## 本幕 takeaway 应落到
- 正则化回归不是另起炉灶，而是在原有拟合目标中加入复杂度约束；
- 所有后续方法都可以看成“损失函数不变，惩罚方式不同”；
- 所谓方法差异，本质上是“我们允许模型怎样被约束”。

## 教师备注
- 这里可以先不展开优化算法；
- 不建议把 Bias-Variance 曲线上的动点作为本幕主图，它太抽象，不利于承接后面的 path 图；
- 语言上要反复强调：`penalty` 不是为了让模型更差，而是为了让模型更可控。

# 幕三：Ridge vs Lasso——同样收缩，为什么一个保留全部、一个开始筛选？

## 本幕定位
这是本节课视觉冲击最强的一幕。
建议用 regularization path 作为主舞台，让学生直接“看见” `Ridge` 和 `Lasso` 的差异。

## 核心冲突
> 同样是在给系数施加约束，为什么 `Ridge` 倾向于让所有变量一起缩小，而 `Lasso` 却会把一些变量直接压成 0？

## 本幕主图
**`Ridge` 与 `Lasso` 的 regularization path 并排对比图**

这张图是本周最重要的主图之一。
它要让学生一眼看出：
- `Ridge` 更像“大家一起缩小”；
- `Lasso` 更像“有些变量被挤出场”；
- 在生产数据时，特意给出 3 列有强烈共线性；
- 同样调 `alpha`，不同 penalty 会产生完全不同的路径行为。

主图要服务的不是数学形式，而是“模型性格”的可视化对照。

## 本幕辅图
**Bias-Variance 权衡中的动图或稳定性辅助图**

这张辅图的任务不是重复 path，而是回答另一个问题：
- 当 penalty 逐渐变大时，模型为什么会变得更稳？
- 我们到底是拿什么，交换到了更小的 variance？

这张辅图要帮助学生把“系数路径变化”连接回“泛化与稳定性”的主线。
它更适合作为动态演示或课堂口播配合图，而不必承担精确定量比较的任务。

## 本幕要完成的认知分化
### Ridge
- 关键词：收缩、平滑、稳定、抗共线性；
- 典型表现：系数随 `alpha` 增大而逐步缩小，但通常不直接变成 0；
- 适合强调：如果主要目标是预测稳定性，Ridge 很有吸引力。

### Lasso
- 关键词：稀疏、筛选、变量名单收缩；
- 典型表现：随着 `alpha` 增大，部分系数会被压到 0；
- 适合强调：Lasso 提供了“模型自动筛选变量”的可见现象。

## 围绕图的课堂问题
- 如果你只能看这两张路径图，你怎么判断哪个方法更像“筛选器”？
- 你觉得“系数越来越小”和“变量直接消失”在业务上意味着什么不同？
- 如果你要交一份更短的变量名单给业务方，你会更偏向哪一种？

## 本幕 takeaway 应落到
- `Ridge` 的核心价值不在筛选，而在稳定收缩；
- `Lasso` 的核心价值在于把“变量筛选”做成了优化结果的一部分；
- 同样叫正则化，不同 penalty 会导向完全不同的模型行为。

## 教师备注
- 这一幕不要急着讲太多高维理论，先让学生“看出差异”；
- 不建议只展示 `\beta_1, \beta_2` 两个系数，否则“变量名单变化”的戏剧性不够；
- 可以明确说：今天不是要背定义，而是要学会根据任务目标选工具。

# 幕四：Elastic Net——当变量高度相关时，折中为什么更成熟？

## 本幕定位
这一幕的任务，是避免课堂停留在“Ridge vs Lasso 二选一”的扁平对立里。
要让学生看到：现实数据里，特征往往不是彼此独立的，相关变量成组出现时，纯 `Lasso` 的筛选可能不够稳。

## 核心冲突
> 如果几个变量本质上传递的是相近信息，Lasso 只留下其中一个，真的总是合理吗？

## 本幕主图
**`Lasso` 与 `Elastic Net` 在相关变量场景下的路径对比图**

这张图要直接回答：
> 当变量成组相关时，模型到底是在“选代表”，还是在“让一组变量共同保留并收缩”？

主图要教会学生：
- 纯 `Lasso` 在相关变量下可能显得过于武断；
- `Elastic Net` 更适合处理“既想筛选，又不想丢掉整组相关信息”的情形；
- 方法选择要回到数据结构，而不是只看谁更稀疏。

## 本幕辅图
**Elastic Net 参数角色的辅助说明图或小表**

这张辅图只承担说明作用：
- 一个参数控制总体惩罚强度；
- 一个参数控制更偏向 `Lasso` 还是更偏向 `Ridge`。

它的目的不是扩展复杂公式，而是防止学生把 Elastic Net 误解成“神秘第四种黑箱方法”。

## 本幕必须完成的认知任务
- 说明 `Elastic Net` 不是“为了多一个名词”，而是为了解决相关特征场景下的实际问题；
- 让学生看到：它结合了 `L1 + L2`，既保留筛选能力，又改善纯 Lasso 在相关变量下的激进性；
- 建立一个现实判断：相关变量成团时，Elastic Net 往往更稳健。

## 围绕图的课堂问题
- 如果两列变量几乎表达的是同一件事，模型只留一个，会不会太武断？
- 你觉得业务上更需要“一份最短名单”，还是“一份更稳的名单”？
- 当变量之间明显相关时，为什么纯粹追求稀疏可能反而带来不稳定？

## 本幕 takeaway 应落到
- `Elastic Net` 适合处理“既想筛选，又不想在相关变量面前过于武断”的场景；
- 当特征高度相关时，方法选择不能只看“谁更稀疏”，还要看“谁更稳”；
- 三种方法的比较标准，应该回到任务目标与数据结构。

## 教师备注
- 这一幕是方法成熟度的一幕，不是再加一个名词定义；
- 要帮助学生形成“相关变量成组”这个现实感；
- 不建议在这里再额外展开新的几何图主线，以免视觉系统过多。

# 幕五：会用比会看更重要——`GridSearchCV` 如何帮我们选 `alpha`？

## 本幕定位
这是最后的“模型选择判断”收束幕。
目的不是再加新算法，而是把学生从“看图很有意思”带回“到底怎么做选择”。

## 核心冲突
> 看懂路径图很容易，但 `alpha` 到底应该怎么选？是越大越好吗，还是越稀疏越好吗？

## 本幕主图
**`GridSearchCV` / CV 分数随 `alpha` 变化的主图**

这张图要回答的是：
> `alpha` 到底应该怎么选？是越大越好吗，还是越稀疏越好吗？

它要教会学生：
- 正则化强度不能靠肉眼拍脑袋；
- 最稀疏的模型不一定最好；
- 模型选择应该回到验证表现，而不是只看路径图是否“好看”。

## 本幕辅图
**最终模型对比结果表（可选）**

这张辅图如果保留，只需要承担总结作用：
- 哪个模型更偏向稳定预测；
- 哪个模型更偏向变量筛选；
- 哪个模型在相关变量场景下更稳健。

## 本幕必须完成的三个判断
1. `alpha` 不能靠肉眼挑，要通过 `GridSearchCV` 等交叉验证来选；
2. 最稀疏不一定最好，最强惩罚也不一定最好；
3. `variable selection` 是预测建模工具里的筛选结果，不应自动升级为因果解释结论。

## 围绕图的课堂问题
- 最稀疏的模型一定是最好的模型吗？
- 如果某个 `alpha` 让变量数量最少，我们就应该选它吗？
- Lasso 选中了一个变量，我们能不能立刻说“它就是最重要原因”？

## 本幕 takeaway 应落到
- 正则化方法的价值，不只在于图像漂亮，还在于能不能通过验证结果支持模型选择；
- `alpha` 的选择本质上是泛化表现的比较，而不是视觉上的偏好；
- `variable selection` 是预测建模工具里的筛选结果，不应自动升级为因果解释结论。

## 教师备注
- 这幕要有“降温”效果：把学生从技术兴奋拉回模型选择判断；
- 这幕不再强调 `Pipeline` 和 `data leakage`，默认学生已在之前练习中较充分接触；
- 如果时间紧，这幕优先保留 `CV` 主图，不必贪多。

# Synthesis 设计建议

## Synthesis 1（幕三之后）
建议放在 `Ridge vs Lasso` 之后。

### 作用
帮助学生先收住前半节的核心对照：
- 为什么会有正则化；
- 什么是 `loss + penalty`；
- Ridge 与 Lasso 的行为差异。

### 建议一句话锚点
> 到这里，学生应能理解：正则化不是“多学三个名词”，而是在同一个优化框架下，用不同 penalty 交换不同的模型性格。

---

## Synthesis 2（全课结尾）
建议放在幕五之后。

### 作用
把全课的判断压缩成一个可复述框架。

### 建议收束框架
- 想要稳定收缩、抗共线性：优先想到 `Ridge`；
- 想要稀疏结果、自动筛选：优先想到 `Lasso`；
- 遇到相关变量成组、又想筛选又想稳：优先考虑 `Elastic Net`；
- 无论哪一种，最终都要回到 `CV` / `GridSearchCV` 的验证结果做选择；
- 被选中的变量不自动拥有因果解释地位。

# Transition 设计建议

## 下一个问题应该怎么自然提出？
本周最后不应停在“已经会三种正则化”，而应留下一个自然开放问题：

> 如果我们不再只追求“把变量筛得更少”，而是开始追求“如何评价模型在不同数据上的泛化表现”，那么模型选择的标准还能怎样系统化？

或者更口语化一些：

> 现在我们已经知道模型不能只看训练集拟合，那接下来，一个模型到底该怎样被公平比较？

这个 transition 可以自然衔接到：
- 更系统的模型评估；
- 偏差—方差权衡的回看；
- 交叉验证的进一步应用；
- 或后续更广义的模型选择主题。

# 后续生成 notebook 时的实现提醒

## 1. 场景数量
本设计稿默认生成：
- 5 个核心 scene；
- 2 个 synthesis；
- 1 个 transition。

## 2. 每幕内部默认节奏
每一幕尽量按以下 rhythm 展开：

`prompt → stage → takeaway`

必要时可在 `stage` 前加一个 setup code cell，但不要让单幕代码过长。

## 3. 图形与表格优先级
优先保证以下输出可读：
- OLS 不稳定或系数波动的主图；
- `loss + penalty` 的结构主图；
- `Ridge` / `Lasso` regularization path 的并排主图；
- 体现“penalty 变大如何换来更稳定模型”的辅图或动图；
- `Elastic Net` 与 `Lasso` 在相关变量场景下的对比主图；
- `GridSearchCV` / `CV` 结果主图；
- 必要时再补简洁结果表。

## 3.1 图形组织原则
后续生成 notebook 时，每一幕尽量围绕“主图 + 辅图”组织，而不是围绕“概念定义”组织。

要求：
- 每一幕先想清楚：**这张主图要让学生学会什么判断？**
- prompt 要优先对图发问，而不是先对术语发问；
- takeaway 要优先回答“这张图说明了什么”，再上升到方法选择；
- 辅图只负责支撑主图，不额外开新的认知主线；
- 如果一张图不能清楚回答一个问题，就不要让它承担本幕主图角色。

## 4. 本周最不能丢的教学句子
后续写 notebook 时，至少要把以下判断讲清：
- 正则化是在 loss 外显式加入复杂度成本；
- `Ridge` 强在稳定收缩，不强在筛选；
- `Lasso` 强在稀疏与变量筛选；
- `Elastic Net` 强在相关变量场景下的折中稳健；
- 变量筛选不等于因果解释；
- scaling 放在 CV 外属于高风险错误。

## 5. 代码实践必须覆盖的规范动作
- 使用固定随机种子；
- 使用 `GridSearchCV` 或等价 CV 选参；
- 至少生成一张能够支持模型选择判断的 `CV` 主图；
- 在方法比较里明确区分“稳定预测”“变量筛选”“相关变量场景下的折中稳健”。
