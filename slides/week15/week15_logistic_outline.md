# Week 15 Notebook 分幕设计稿

## 主题
广义线性模型（GLM）核心：`Logistic Regression`

## 设计定位
这周不把逻辑回归讲成“在线性回归后面接一个 `sigmoid` 就好了”。
更自然的叙事应该是：

> 当响应变量不再是连续数值，而是 `0/1` 分类结果时，我们还能不能保留“线性预测”这条主线？如果能，代价是什么，改变又发生在哪里？

课堂希望学生最终形成三条判断：
- 逻辑回归不是“普通线性回归直接套一个激活函数”；
- 逻辑回归真正建模的是概率、`log-odds` 和 `Bernoulli likelihood`；
- 逻辑回归之所以长期不过时，不只是因为简单，而是因为它在解释性、稳定性、概率输出和工业可部署性之间达到了很好的平衡。

---

## 总叙事主线
本节课建议围绕下面这条线推进：

1. 先让学生看到：如果把 `0/1` 分类问题直接当作普通回归来做，会出现哪些明显的不自然；
2. 再把模型从“预测一个实数”推进到“预测一个概率”，让 `sigmoid` 出场；
3. 接着说明：一旦输出变成概率，我们自然要用 `Bernoulli` 和 `MLE` 来写目标函数，而不是继续最小化平方误差；
4. 然后引出 `log-odds`，把系数的统计解释和业务解释真正立住；
5. 最后再讨论：为什么工业界仍然大量使用逻辑回归，以及正则化逻辑回归在高维分类中的位置。

# 幕一：分类问题为什么不能直接拿 `OLS` 硬做？

## 本幕定位
这是整节课的 stakes 幕。
重点不是先定义逻辑回归，而是先制造一个明确的不适：

- 响应变量明明只有 `0/1`；
- 普通线性回归却会预测出 `<0` 或 `>1` 的值；
- 而且误差结构和概率解释都开始变得别扭。

## 本幕主图
**一维或二维二分类数据上，`OLS` 拟合线与预测值散点图**

主图要让学生直接看见：
- 线性回归输出的是实数，不天然受限于 `[0, 1]`；
- 把它硬解释成概率会出问题；
- 分类任务和普通回归的响应变量结构并不相同。

## 围绕图的课堂问题
- 如果模型给出 `1.18` 或 `-0.23`，你还能把它解释成概率吗？
- 即使 `OLS` 在某些局部看起来能分开两类，它到底在优化什么？
- 既然标签只有 `0/1`，我们是不是应该直接建模“属于 1 的概率”？

## 本幕必须补的严格对象
- 二分类响应变量：`Y \in {0,1}`
- 普通线性回归的形式：`E[Y|X] = X\beta` 在这里会遇到边界问题

## 这里，大家需要记住的是
- 分类任务不是不能保留“线性预测”这条思路；
- 但不能继续把输出直接当作一个无边界实数；
- 所以逻辑回归的第一步，不是换掉线性部分，而是换掉“输出如何被解释”。

# 幕二：如果要预测概率，为什么 `sigmoid` 很自然？

## 本幕定位
这一幕负责让 `sigmoid` 出场，但不要把它讲成“神秘函数”。
重点是回答：

> 如果线性部分仍然想保留为 `\eta = X\beta`，那怎样把它稳定地映射到 `[0,1]`？

## 本幕主图
**`sigmoid` 曲线 + 若干线性预测值映射到概率的示意图**

这张图要教会学生：
- 线性预测量 `\eta` 对输入没有额外限制；
- `sigmoid(\eta)` 会把它压到 `[0,1]`；
- 这时模型输出更像“概率”，而不是随便一个数。

## 围绕图的课堂问题
- 为什么把任意实数压到 `[0,1]` 是这里的核心需求？
- 当 `\eta` 很大或很小时，概率会发生什么？
- `sigmoid` 的中点 `0.5` 和线性预测量 `0` 之间有什么对应关系？

## 本幕必须补的严格对象
- 线性预测子：`eta = X\beta`
- `sigmoid`：`p(x) = 1 / (1 + e^{-eta})`

并且建议把 `sigmoid` 的定义放在主图之前，而不是等讲完图像性质再给公式。

## 这里，大家需要记住的是
- 逻辑回归并没有抛弃线性结构；
- 它是在“线性预测量”外面，加了一个把输出压成概率的 link；
- 所以逻辑回归更准确的说法是：**线性预测 + 概率映射**。

# 幕三：为什么这里自然会走到 `Bernoulli + MLE`？

## 本幕定位
这一幕是全课最关键的“从直观到统计建模”的过桥幕。
重点是让学生意识到：

- 一旦输出被解释为概率；
- 响应变量又只有 `0/1`；
- 那就应该自然转向 `Bernoulli` 分布和极大似然估计，而不是继续沿用平方误差。

建议先加一个子问题：

> 既然标签还是 `0/1`，为什么不继续用 `MSE / RMSE` 作为损失函数？

这样学生才会真正理解：我们不是“随手换一个 loss”，而是在换统计建模对象。

## 本幕主图
**同一个样本在不同预测概率下，平方误差与负对数似然的对比图**

建议用两种样本类型并排比较：
- 真值 `y = 1` 时，随着预测概率增加，likelihood 如何变化；
- 真值 `y = 0` 时，随着预测概率增加，likelihood 如何变化。

主图要让学生看见：
- 平方误差当然也会惩罚错误；
- 但它对“错得很自信”的惩罚不如负对数似然敏感；
- 一旦我们承认响应变量服从 `Bernoulli`，`log loss` 就不再是凭空指定，而是由 `MLE` 自然推出。

## 围绕图的课堂问题
- 如果一个样本真实标签是 `1`，模型却只给出 `0.02` 的概率，这种错误为什么比“给 0.45”更严重？
- 为什么分类任务里“错得很自信”要被重罚？
- 既然模型输出的是概率，我们为什么不直接比较 likelihood？

## 本幕必须补的严格对象
- `Y ~ Bernoulli(p)`
- 单样本 likelihood：`p^y (1-p)^(1-y)`
- 对数似然 / log loss 形式

这些定义建议出现在主图之前，而不是最后才揭示。

## 这里，大家需要记住的是
- 逻辑回归不是“先回归，再凑一个分类阈值”；
- 它从一开始就在建模 `Bernoulli` 概率；
- 所以它的训练目标来自 `MLE`，而不是普通回归里的平方损失。

# 幕四：`log-odds` 到底是什么，系数为什么能解释？

## 本幕定位
这是逻辑回归最值得保留的解释性一幕。
这幕不只是讲变换公式，而是要让学生真正掌握：

> 为什么逻辑回归的系数解释，落脚点在 `log-odds` 而不是“概率直接加几”？

建议不要先讲图像和性质，再最后才给定义。
更自然的顺序应该是：

1. 先定义 `odds`；
2. 再定义 `logit(p)`；
3. 再讨论图像、性质和系数解释。

## 本幕主图
**概率、odds、log-odds 对照表 / 曲线图**

可以考虑以下两种展示方式之一：
- 一张从 `p -> odds -> log-odds` 的对照表；
- 或者一张概率与 log-odds 对应关系曲线图。

主图要教会学生：
- 概率不是线性变化的；
- `log-odds` 才和线性预测量 `X\beta` 对接；
- 系数解释落在“自变量增加 1 单位，log-odds 如何变化”。

## 围绕图的课堂问题
- 为什么逻辑回归不直接说“`x` 增加 1，概率增加多少”？
- `odds = p/(1-p)` 在业务上该怎么理解？
- 如果某个系数是正的，意味着什么？如果是负的呢？

## 本幕必须补的严格对象
- `odds = p / (1-p)`
- `logit(p) = log(p/(1-p)) = X\beta`
- 系数、odds ratio 与 `exp(beta_j)` 的联系

## 这里，大家需要记住的是
- 逻辑回归真正线性的是 `log-odds`，不是概率本身；
- 因此系数的统计解释是：自变量变化如何影响 `log-odds`；
- 对业务方来说，更自然的翻译通常是：某个变量会让“发生的相对可能性”上升还是下降。

# 幕五：分类指标为什么不能只看 `accuracy`？

## 本幕定位
这一幕把模型从“会不会拟合”推进到“如何评估分类表现”。
重点是让学生意识到：

- 分类问题不是只有“分对多少”；
- 指标背后对应的是不同业务偏好；
- 概率输出模型尤其适合讨论阈值、召回率、ROC-AUC、log loss。

建议这里也先给定义，再看图：

1. 先给混淆矩阵 `TP / TN / FP / FN`；
2. 再给 `accuracy / precision / recall / F1`；
3. 最后再用图去讨论 threshold、ROC-AUC、log loss。

## 本幕主图
**同一二分类案例下，不同 threshold / 不同模型的指标对比图**

建议至少让学生看到以下两类现象中的一种：
- threshold 改变时，recall / precision / accuracy 如何变化；
- 两个模型 accuracy 接近，但 ROC-AUC 或 log loss 明显不同。

## 围绕图的课堂问题
- 为什么 `accuracy` 高不一定代表模型真的好？
- 如果我们在做违约预测或流失预警，为什么 recall 往往比 accuracy 更敏感？
- `log loss` 相比 accuracy，额外惩罚了什么？

## 本幕必须补的严格对象
- 混淆矩阵
- `accuracy`
- `precision`
- `recall`
- `ROC-AUC`
- `F1`
- `log loss`

## 这里，大家需要记住的是
- 分类指标不是彼此替代，而是在回答不同问题；
- 逻辑回归的概率输出，使它非常适合和阈值、风险偏好、概率损失联系起来；
- 所以评估分类器时，不能只盯着“分对了多少个”。

# 幕六：正则化逻辑回归为什么在工业界一直不过时？

## 本幕定位
这一幕负责把 week13 和本周真正接起来。
重点不是再讲一遍 `L1/L2`，而是要让学生看到：

- 逻辑回归也可以正则化；
- 当特征很多、分类边界容易过拟合时，`L1/L2` 同样重要；
- 逻辑回归的工业生命力，来自它在解释性、稳定性、速度、概率输出之间的平衡。

## 本幕主图
**未正则化 vs `L1/L2` 正则化逻辑回归在高维分类上的系数或指标对比图**

这张图要回答：
- `L1` 更偏向稀疏筛选；
- `L2` 更偏向平滑收缩；
- 即便在分类任务中，正则化也仍然是“稳定性工具”。

## 围绕图的课堂问题
- 为什么高维分类里，逻辑回归也会需要正则化？
- `L1` 和 `L2` 在逻辑回归中的“性格”是否和在线性回归里类似？
- 如果工业界已经有很多更复杂的模型，为什么逻辑回归还没退场？

## 本幕必须补的严格对象
- 带 penalty 的 logistic objective
- `L1` / `L2` 对高维分类与特征筛选的作用

## 这里，大家需要记住的是
- 逻辑回归不是过时模型，而是非常强的 baseline 和解释工具；
- `L1` 更适合稀疏筛选，`L2` 更适合稳定收缩；
- 在很多工业场景里，“可解释 + 可校准 + 可快速部署”本身就是重要优势。

## 结尾收束
建议最终用一句话收回全课：

> 逻辑回归保留了“线性预测”这条主线，但把输出、分布假设和目标函数都改造成了适合二分类概率建模的形式。

并自然抛出下一个问题：

> 如果 `0/1` 标签对应的是 `Bernoulli + logit`，那当响应变量变成计数、比例或多类别时，GLM 又会怎样继续扩展？

这样可以自然引向：
- 更一般的 `GLM`
- `Poisson regression`
- 多分类逻辑回归
- 或者后续更复杂分类模型。
