# 模拟数据与真实二手车数据回归建模对照总结

## 1. 知晓DGP前提下，模拟数据结果推测更简单的原因
DGP即数据生成过程，模拟数据集的特征作用方向、变量权重、噪声大小、共线性关系、缺失值与异常值分布均由人工预先定义。我们完全清楚价格的真实构造公式，能够明确每一项特征对目标变量的真实影响大小与正负趋势，所有数据问题都是人为主动搭建而成。建模完成后可以直接对照既定生成规则判断模型拟合好坏、系数偏移原因，快速区分模型拟合偏差与数据自带扰动，不需要反向挖掘潜在规律，因此实验分析与结果预判难度极低。

## 2. 真实数据模型指标尚可，但业务解释难度更高的原因
第一，真实业务数据不存在固定的数据生成公式，样本中存在大量未录入的隐性影响因素，例如车辆实际车况、维修记录、地域行情、市场热度等，无法全部纳入建模特征；
第二，特征之间不仅存在线性关系，还包含复杂隐性交互与非线性关联，普通线性回归仅能捕捉表层趋势，难以贴合真实作用逻辑；
第三，各类特征存在混杂效应，单一变量往往叠加多重业务含义，无法独立判定其对价格的纯粹影响；
第四，模型学习到的是市场整体统计规律，并非严格因果关系，仅能实现趋势预测，很难完成严谨、合理的业务层面归因解释。

## 3. 共线性、缺失值、异常值在两类数据集上的影响差异
### （1）多重共线性
模拟数据：缺失值为人工随机生成，集中在 engine_power和 price 这两个数值特征上，缺失规律清晰固定，对整体数据分布几乎没有影响。对 engine_power 和目标变量 price 均使用中位数填补。
真实数据：共线性来源十分繁杂，不仅连续变量存在相关性，独热编码生成的大量虚拟变量极易产生高程度共线性，大范围干扰全体特征系数，须依靠Ridge正则化进行约束修正。

### （2）缺失值
模拟数据：缺失值为人工随机生成，缺失规律清晰固定，对整体数据分布几乎没有影响。 用中位数填补数值型特征，用众数填补类别型特征。
真实数据：本次使用的二手车数据无任何缺失。为保证代码通用性，仍对数值变量使用中位数填补、类别变量使用众数填补。

### （3）异常值
模拟数据：异常值为统一批量生成，数量固定、形态简单，仅小幅拉高预测误差，不会改变数据集整体价格分布趋势。
真实数据：异常值来源杂乱，包含天价豪车、报废低价车、平台录入错误数据等，极端数值跨度极大，不仅会大幅拉高模型误差，还会直接扭曲特征与目标变量之间的基础拟合关系。

## 4. 无泄露交叉验证在真实数据建模中尤为重要的原因
1. 真实二手车数据存在明显分层结构，按照车型、价位、使用年限自然分层，简单随机划分训练集与测试集容易出现分布错位；
2. 真实数据预处理流程复杂，缩尾统计阈值、标准化均值方差都依赖数据整体分布，若提前使用全量数据拟合预处理规则，会直接将测试集信息泄露至训练流程，得到虚高、不真实的模型效果；
3. 真实建模以业务落地为目标，无泄露五折交叉验证能够严格遵循工业界建模规范，客观还原模型在未知新样本上的真实泛化能力，有效规避过拟合与数据泄露带来的实验假象，保证实验结果具备实际参考价值。而模拟数据分布均匀规整，数据泄露对最终实验结果影响极小。

## 5. 自研utils工具库节省的重复开发工作
1. 封装自定义标准化工具`CustomStandardScaler`，统一实现训练集拟合、测试集转换逻辑，无需重复编写数据标准化代码；
2. 封装VIF多重共线性诊断函数，模拟数据与真实数据实验可直接调用，省去重复编写共线性判定与结果打印的冗余代码；
3. 统一封装自研OLS普通线性回归、Ridge正则化回归模型，固定训练与预测调用接口，摆脱第三方库依赖，简化模型切换流程；
4. 集中封装RMSE、MAE、MAPE、R²四类回归评估指标，两份实验共用同一套评估逻辑，避免手动编写公式出现计算错误，统一结果输出格式；
5. 搭建标准化建模实验流程，固定数据读取、清洗、预处理、建模、验证、结果输出整套范式，减少代码调试时间，让实验重心聚焦在数据分析与结论总结上。

## 整体建模差异总结
1. 可控性：模拟数据全流程可控，适合学习回归算法原理；真实数据不可控，高度贴合职场实战数据分析场景；
2. 建模重心：模拟数据以验证算法原理、对照DGP校验拟合效果为主；真实数据以脏数据清洗、偏差修正、正则化优化、业务拟合为主；
3. 优化方向：模拟数据简单预处理即可实现较高拟合精度；真实数据必须搭配异常值处理、特征衍生、正则化约束、严谨交叉验证等多种手段提升模型稳定性；
4. 适用场景：模拟数据用于课堂理论学习、算法原理验证；真实数据用于实战项目练习、企业级预测模型搭建训练。
