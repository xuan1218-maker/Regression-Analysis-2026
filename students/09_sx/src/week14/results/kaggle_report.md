# 真实数据挑战报告

## 1. 数据来源与预处理

### 1.1 数据集
- **来源**: Kaggle House Prices: Advanced Regression Techniques
- **原始维度**: 1460样本 × 81特征
- **目标变量**: SalePrice（房屋售价）

### 1.2 关键变量选择

**选择方法**: 保留所有数值型特征 + 对类别型特征进行One-Hot编码

**选择理由**:
1. 数值型特征（如LotArea, TotalBsmtSF, GrLivArea等）直接反映房屋物理属性
2. 类别型特征（如Neighborhood, MSZoning等）反映位置和分区信息
3. One-Hot编码保留了类别信息的全部维度，避免信息损失

### 1.3 预处理步骤
1. 移除缺失率>50%的特征: 5个
2. 数值型特征：中位数填补缺失值
3. 类别型特征：众数填补 + One-Hot编码
4. 对目标变量取对数（处理偏态分布）

### 1.4 清洗后数据
- **样本量**: 1460
- **特征数**: 233

## 2. 模型表现对比

| 模型 | Test RMSE | 备注 |
|------|-----------|------|
| OLS | 0.1835 | 条件数=inf, rank=225/233 |
| Lasso | 0.1395 | 非零系数=52 |
| PCR | 0.1498 | 最佳k（自动CV选择）=15 |

## 3. 结构判断

### 3.1 OLS不稳定迹象

- **条件数**: inf (inf表示矩阵奇异)
- **秩亏损**: 225/233（特征存在严重共线性）
- **训练-测试差距**: Train RMSE=0.0912, Test RMSE=0.1835

### 3.2 Lasso vs PCR 表现

根据实验结果：
- Lasso 选择保留 52 个非零系数
- PCR 自动选择 15 个主成分

### 3.3 数据结构判断

**结论**: 更接近 Sparse Truth（只有少数特征真正重要）

## 4. 业务解释

如果要向业务方解释这份数据：

> 这份数据更适合 **筛选（selection）** 路线，因为只有一部分特征对房价有显著影响。
> Lasso方法能够识别出这些关键特征，给出简洁的解释。

### 4.1 为什么Lasso更合适？

1. 房屋价格通常由少数核心因素决定（面积、位置、质量）
2. 业务方需要知道"哪些因素最重要"
3. 可解释性在房价预测中至关重要
