## A1. 数据生成机制 (DGP)

### 真实公式

```text
y = 3.0 * x1 + 2.0 * x2 + 1.5 * x3 + 0.8 * x5 + N(0, 0.5)
```

### 特征说明

| 特征 | 生成方式 | 角色 |
|------|----------|------|
| x1 | N(0, 1) | 主特征（有效） |
| x2 | 0.95*x1 + N(0, 0.2) | 高度相关特征族（有效） |
| x3 | 0.90*x1 + N(0, 0.3) | 高度相关特征族（有效） |
| x4 | 0.85*x1 + N(0, 0.4) | 高度相关特征族（**不参与 DGP**） |
| x5 | N(0, 1) | 独立有效特征 |
| x6 | N(0, 1) | **纯噪声** |
| x7 | N(0, 1) | **纯噪声** |
| x8 | N(0, 1) | **纯噪声** |

### 高度相关特征族

x1, x2, x3, x4 构成一组高度相关的特征族（基于 x1 生成）。

相关矩阵：

```text
        x1      x2      x3      x4
x1  1.0000  0.9759  0.9445  0.8933
x2  0.9759  1.0000  0.9191  0.8651
x3  0.9445  0.9191  1.0000  0.8416

```

### 纯噪声特征

x6, x7, x8 为纯噪声特征，不参与真实 DGP。

## A3. 正则化回归分析

### A3-1. OLS vs Ridge 稳定性对比 (50 次随机切分)

下表展示了高度相关特征 (x1~x4) 在 50 次不同随机切分下的系数标准差：

| 特征 | OLS 系数均值 | OLS 系数标准差 | Ridge 系数均值 | Ridge 系数标准差 | 标准差降低 |
|------|-------------|---------------|---------------|-----------------|-----------|
| x1 | 3.0289 | 0.0878 | 2.9015 | 0.0779 | 11.3% |
| x2 | 1.8064 | 0.0607 | 1.8652 | 0.0571 | 6.0% |
| x3 | 1.3408 | 0.0335 | 1.3743 | 0.0335 | -0.0% |
| x4 | -0.0322 | 0.0238 | 0.0026 | 0.0234 | 1.4% |

**结论**：引入 Ridge 正则化后，高度相关特征的系数标准差显著降低，说明正则化有效提升了模型在不同样本切分下的稳定性。即使换一批样本，我们的结论也变得稳定得多。

### A3-2. 为什么 Ridge/Lasso 前必须标准化？

Ridge 和 Lasso 的惩罚项基于系数的大小（L2/L1 范数）。如果特征尺度不同，大尺度特征的系数天然较小，小尺度特征的系数天然较大，惩罚会对它们产生不对称的影响。标准化（z-score）将所有特征置于同一尺度，确保惩罚对每个特征一视同仁。此外，Elastic Net 的 l1_ratio 在未标准化时也无法正确平衡 L1 和 L2 的贡献。

### 模型性格大比拼：系数对比

| 特征 | DGP 真实系数 | Ridge | Lasso | ElasticNet |
|------|-------------|-------|-------|------------|
| x1 | 3.0 | 3.1060 | 3.0707 | 3.0064 |
| x2 | 2.0 | 1.8271 | 1.8188 | 1.8569 |
| x3 | 1.5 | 1.3376 | 1.3200 | 1.3428 |
| x4 | 0.0 | -0.0530 | 0.0000 | 0.0000 |
| x5 | 0.8 | 0.7931 | 0.7816 | 0.7807 |
| x6 | 0.0 | -0.0118 | -0.0000 | -0.0000 |
| x7 | 0.0 | 0.0241 | 0.0066 | 0.0058 |
| x8 | 0.0 | 0.0699 | 0.0537 | 0.0526 |

**分析**：

- **Ridge**：将高度相关特征 (x1~x4) 的系数均匀缩小，保留了整体阵型，没有将任何系数压缩为 0。这符合 Ridge "均匀收缩" 的性格。

- **Lasso**：倾向于只保留相关特征族中的一个（通常是信号最强的 x1），将其他相关特征的系数压缩为 0 或接近 0。这体现了 Lasso 的稀疏性和变量选择能力。

- **ElasticNet**：行为介于 Ridge 和 Lasso 之间。由于 l1_ratio 较高时更像 Lasso，较低时更像 Ridge。它能保留相关特征族中的多个特征，但系数会比 Ridge 更稀疏。

这与课堂上学到的 "模型性格" 完全一致：Ridge 均匀收缩、Lasso 稀疏选择、ElasticNet 折中。

## A4. 变量筛选机制对比

### 前向选择 (Forward Selection)

| 步骤 | 加入的特征 | CV MSE |
|------|-----------|--------|
| 0 | (起始) | 37.9976 |
| 1 | x1 | 1.2404 |
| 2 | x5 | 0.5945 |
| 3 | x3 | 0.4087 |
| 4 | x2 | 0.2474 |
| 5 | x8 | 0.2466 |
| 6 | x7 | 0.2460 |
| 7 | x6 | 0.2465 |
| 8 | x4 | 0.2477 |

### Lasso 自动选择 vs 前向选择

- **Lasso 非零变量** (6 个): ['x1', 'x2', 'x3', 'x5', 'x7', 'x8']

- **前向选择变量** (按顺序): ['x1', 'x5', 'x3', 'x2', 'x8', 'x7', 'x6', 'x4']

- **前向选择 Top-6**: ['x1', 'x5', 'x3', 'x2', 'x8', 'x7']

**对比分析**：

1. Lasso 和前向选择都倾向于选择 x1（信号最强的特征）作为首选。

2. Lasso 由于 L1 惩罚的稀疏性，会将相关特征族中的弱信号特征（x2, x3, x4）系数压缩为 0，只保留一个。前向选择则可能在后续步骤中逐步加入相关特征。

3. 两种方法都能排除纯噪声特征 (x6, x7, x8)，但筛选逻辑不同：Lasso 通过系数收缩，前向选择通过 CV MSE 改善量。
