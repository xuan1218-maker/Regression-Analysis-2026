# 合成数据回归报告

## 1. 数据生成过程（DGP）

**场景**：房屋总价（万元）预测，使用人为构造的特征。

**样本量**：400 行

**特征**：area, rooms, age, location_score, has_garage


### 真实 DGP 公式：

```

total_price = 30 + 0.5*area + 5*rooms - 1.5*age + 3*location_score + 15*has_garage + ε

ε ~ N(0, 10)

```


### 人为注入的数据问题：

- **缺失值**：`rooms` 列的 5% 被设为 NaN

- **异常值**：`age` 列的 2% 被设为 99（极端异常值）

- **共线性**：`location_score = 0.02*area + N(0, 0.5)`，使 `area` 与 `location_score` 高度相关（r ≈ 0.92）

- **尺度差异**：area ~100，age ~15，location_score ~6


### 预期系数方向：

| 特征 | 预期符号 | 原因 |

|------|----------|------|

| area | + | 面积越大的房屋总价越高 |

| rooms | + | 房间越多 → 房价越高 |

| age | − | 房龄越老折旧越多 |

| location_score | + | 地段越好 → 房价越高 |

| has_garage | + | 有车库提升房屋价值 |

## 5. 推断：模型是否恢复了 DGP？

- **area**：系数方向正确（+）。但由于与 location_score 存在共线性，系数可能被衰减。

- **rooms**：方向基本正确。缺失值略微降低了稳定性。

- **age**：如预期为负系数。异常值缩尾处理有助于稳定估计。

- **location_score**：正向但不稳定，因与 area 高度共线。模型难以分离二者的独立效应。

- **has_garage**：如预期呈强正系数。二值特征最容易恢复。


**核心发现**：`area` 与 `location_score` 之间的共线性是主要挑战。两者 VIF > 5 表明模型无法可靠地分离它们的各自贡献，尽管整体预测精度良好。
