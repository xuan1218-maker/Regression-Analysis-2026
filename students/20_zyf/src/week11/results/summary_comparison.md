# 综合对比：合成数据 vs. 真实世界回归

## 1. 为何合成数据的推断更简单？

在处理合成数据时，我们精确知晓 DGP：

- 真实系数、误差分布和特征关系全部可控。

- 我们可以直接将估计系数与真实值进行对比。

- 如果某个系数有偏差，我们可以诊断原因：共线性、噪声还是预处理伪影。

- 相比之下，面对真实数据我们只能说「这就是模型学到的东西」——没有标准答案可供验证。


## 2. 为何真实世界的解释即使在好的指标下也更为困难？

- 真实数据存在未知的混杂因子。我们不知道遗漏了哪些变量。

- 遗漏变量、测量误差或反向因果关系都可能导致系数有偏。

- 例如，Kaggle 模型的 RMSE=25.93 看似合理，但我们无法确认任何单个系数是否「正确」——只能确认预测在平均意义上合理。


## 3. 共线性、缺失值和异常值的影响

| 问题 | 合成数据 | Kaggle 数据 |

|------|----------|-------------|

| 共线性 | 人为设计（area↔location_score）。VIF 明确标记。我们清楚这是真实的共线性。 | 自然存在（如 floor_level↔total_floors）。难以区分是真实关联还是数据伪影。 |

| 缺失值 | rooms 列 5% 完全随机缺失（MCAR），对推断影响小。 | 缺失值可能非随机（MNAR），中位数填补可能引入偏差。 |

| 异常值 | 人为设计（age=99），缩尾处理干净利落。 | 异常值可能是合法的豪宅，也可能是录入错误，难以区分。 |

| 尺度差异 | 已知且通过标准化处理。 | 部分特征尺度极端（如面积 vs 二值标志），标准化至关重要。 |


## 4. 为何无数据泄露的交叉验证对真实数据更为关键

- 在合成数据中，即便发生数据泄露，由于 DGP 简单，仍可能得到「不错」的结果。

- 在真实数据中，数据泄露的后果更为严重：

  - 从全数据集学习到的预处理参数（均值、标准差、填补值）会夸大验证指标。

  - 模型在未见数据上的表现将不如验证结果所示。

  - 在房价预测、医疗、金融等高利害领域尤为危险。

- 我们的实现保证：仅在训练集上调用 imputer.fit() → 验证集上调用 imputer.transform()；仅在训练集上调用 scaler.fit() → 验证集上调用 scaler.transform()。

## 6. 关键收获

1. **合成数据是沙盒**：你可以设计实验来验证你的方法是否真的还原了应该还原的东西。

2. **真实数据令人谦卑**：即使交叉验证分数不错，系数的解读仍需领域知识和审慎态度。

3. **共线性是沉默杀手**：即使预测效果良好，它也会让单个系数变得不可靠。

4. **无泄露预处理不容妥协**：在训练集上拟合、在验证集上变换，是唯一正确的做法。

5. **自建工具库收益显著**：从头编写 transformer、评估指标和模型的初期投入，使后续每个项目都更快、更透明。
