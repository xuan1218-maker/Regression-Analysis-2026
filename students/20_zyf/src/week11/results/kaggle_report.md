# Kaggle 真实数据回归报告

## 1. 数据集信息

- **数据集名称**：青州市二手房/房产数据

- **来源**：Kaggle（sh_data.csv）

- **样本量**：18339 行

- **预处理后特征数**：20

- **目标变量**：`total_price`（总价，单位：万元）—— 连续型

- **业务含义**：每一行代表青州市的一条二手房挂牌记录。


### 为什么选择这个数据集？

1. 它是真实世界的挂牌数据，而非清洗过的玩具数据集。

2. 包含数值型（面积、总价）、类别型（装修、朝向、电梯）和半结构化（楼层描述、户型描述）等多种特征类型。

3. 存在真实的数据质量问题：格式不一致、潜在异常值、高基数类别变量。

4. 房价预测是一个有经济学意义的回归问题，具有清晰的业务解释。


## 2. 预处理摘要

- **删除的列**：`id`（无用）、`title`（文本）、`unit_price`（数据泄露：单价 = 总价 / 面积）、`community`（类别过多）、`nearby`（与区域重复）。

- **特征工程**：从结构化文本字段中提取了楼层高度、总层数、卧室数量和客厅数量。

- **类别编码**：对区域（district）和产权（ownership）进行独热编码；装修等级进行序数编码；电梯、朝向进行二值编码。

- **缺失值填补**：中位数填补（在每折交叉验证内完成，防止数据泄露）。

- **异常值处理**：1%/99% 分位数缩尾处理（在每折交叉验证内完成）。

- **标准化**：Z-score 归一化（在每折交叉验证内完成）。

## 6. 业务解读

### 模型误差的业务含义

- 平均预测误差（MAE）约为 **16.1 万元**。

- 对于一套标价 80 万元的典型住宅，模型预测偏差平均约为 20.1%。


### 哪些变量最稳定？

- `square`（面积）：系数最大且最稳定，符合预期。面积越大的房屋总价越高。

- 建筑类型和区域变量的系数模式相对稳定。


### 哪些变量不确定？

- VIF 较高的特征（如 total_floors、floor_level、bedrooms）因共线性导致系数估计不稳定。

- decoration_level 的系数可能受豪宅异常值的影响。


### 若投入生产可能面临的风险：

1. **数据漂移**：房地产市场条件随时间变化；模型需要定期重新训练。

2. **地域局限性**：该模型仅基于青州市数据训练——将其推广到其他城市无效。

3. **缺少关键特征**：未包含学区、市场走势、经济指标等因素。

4. **异常值敏感性**：极端豪宅或急售房源可能未被充分代表。
