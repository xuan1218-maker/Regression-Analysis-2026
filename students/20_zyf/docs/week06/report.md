## 🛠️ 2. 设计思路与实现方案

### 2.1 编程范式选择：面向对象 vs. 过程式

**选择：面向对象（Class方式）✅**

#### 为什么不选过程式？

| 维度 | 过程式函数 | 面向对象类 |
|------|-----------|----------|
| **状态管理** | 状态散落，参数列表长 | 封装在实例，清晰明确 |
| **多实例场景** | 易传错参数、易互相污染 | 两个市场模型完全独立 |
| **代码可读性** | `procedural_fit(X, y, ...)` | `model.fit(X, y)` |
| **扩展性** | 每次新需求都改函数签名 | 新方法直接加入class |
| **与sklearn兼容** | 需要额外包装适配器 | 天然契合，鸭子类型 |

#### 选择Class带来的好处

```python
# 场景B中的两个独立市场
model_na = CustomOLS()  # 北美模型
model_eu = CustomOLS()  # 欧洲模型

# 各自拥有独立的系数、协方差矩阵、残差方差
model_na.fit(X_na, y_na)  # 互不干扰
model_eu.fit(X_eu, y_eu)
```

这种设计使得双市场独立分析变得优雅而安全。

### 2.2 截距项处理

在原始数据X的第一列插入全1向量，将截距转化为普通系数β₀。这与sklearn中`fit_intercept=True`的效果等价。

```python
X_with_intercept = np.column_stack([np.ones(n), X])  # (n, k+1)矩阵
```

**优点**：统一了系数向量的形式，简化了计算与解释。

### 2.3 鸭子类型与通用评价

核心理念：**不问类型，只问行为**。编写的 `evaluate_model()` 函数既能评价自己的CustomOLS，也能评价sklearn的LinearRegression：

```python
def evaluate_model(model, X_train, y_train, X_test, y_test, model_name: str):
    """只要有 .fit(), .predict(), .score() 就能用"""

    r2 = model.score(X_test, y_test)
    # ...
```

## 🔧 3. 核心算法与实现细节

### 3.1 CustomOLS类的关键方法

#### fit(X, y)：计算参数估计

$$\hat{\beta} = (X^\top X)^{-1}X^\top y$$

计算步骤：
1. 计算 $X^\top X$ 及其逆矩阵
2. 计算 $X^\top y$
3. 求解 $\hat{\beta} = (X^\top X)^{-1}X^\top y$
4. **核心**：保存协方差矩阵 $\text{Cov}(\hat{\beta}) = \hat{\sigma}^2(X^\top X)^{-1}$，为后续F检验做准备

```python
def fit(self, X: np.ndarray, y: np.ndarray):
    XtX_inv = np.linalg.inv(X.T @ X)
    self.coef_ = XtX_inv @ X.T @ y
    
    # 残差方差估计 σ² = SSE / (n - k)
    residuals = y - X @ self.coef_
    self.sigma2_ = (residuals @ residuals) / (len(y) - X.shape[1])
    
    # 协方差矩阵
    self.cov_matrix_ = self.sigma2_ * XtX_inv
    self.df_resid_ = len(y) - X.shape[1]
    return self
```

#### score(X, y)：拟合优度R²

$$R^2 = 1 - \frac{SSE}{SST} = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$$

#### f_test(C, d)：一般线性假设检验

对于假设 $H_0: C\beta = d$，F统计量为：

$$F = \frac{(C\hat{\beta} - d)^\top[C(\text{Cov}(\hat{\beta}))C^\top]^{-1}(C\hat{\beta} - d)}{q \cdot \hat{\sigma}^2}$$

其中：
- $q$ = 约束条件个数（即C的行数）
- $\hat{\sigma}^2$ = 估计的残差方差
- $F \sim F_{q, n-k}$ 分布

**关键优势**：无需重新计算$(X^\top X)^{-1}$，直接复用 `fit()` 中保存的协方差矩阵！

### 5.3 F检验：广告效果联合显著性

#### 假设设定

- **H₀（虚无）**：β_TV = β_Radio = β_Social = 0（广告整体无效）
- **H₁（备择）**：至少有一个广告系数非零（广告有效）
- **显著性水平**：α = 0.05

#### 检验结果

| 市场 | F统计量 | p值 | 结论 |
|------|--------|------|------|
| **北美** | 250.92 | <0.001 | ✅ **强烈拒绝H₀**，广告效果显著 |
| **欧洲** | 719.83 | <0.001 | ✅ **强烈拒绝H₀**，广告效果显著 |

**统计学意义**：
- 两个p值都远小于0.05
- 欧洲的F值(719.83) > 北美(250.92)，说明欧洲市场的广告效果更加显著和稳定

- **结论**：两个市场的模型都达到了极佳的拟合，广告投入与销售的关系被完美刻画

A：在多市场独立分析场景中，Class完美地通过实例化解决了"多状态管理"问题，使得model_na和model_eu能够安全共存，这是过程式函数无法优雅实现的。

这反映了两个市场的消费者特征与媒体生态的本质差异。

**Q3：如何确保截距项处理的正确性？**

A：通过向X矩阵插入全1列，将截距转化为β₀这个普通系数。在合成数据测试中，CustomOLS与sklearn的R²完全一致，证明了这种处理的正确性。
