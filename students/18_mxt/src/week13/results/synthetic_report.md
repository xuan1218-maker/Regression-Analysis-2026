# 共线性模拟数据实验报告（Task A）

## 一、数据生成机制（DGP）
1. 样本量：500
2. 特征数量：8 个（feat_1 ~ feat_8）
3. 真实生成规则：
   \( y = 2.5 \times \text{feat\_1} + 1.8 \times \text{feat\_4} + \mathcal{N}(0, 1.2) \)
4. 高度相关特征组：feat_1、feat_2、feat_3（人工构造强相关）
5. 真实有效特征：feat_1、feat_4
6. 纯噪声特征：feat_7、feat_8

## 二、正则化前标准化的必要性
Ridge、Lasso 均以系数大小作为正则惩罚依据，不同特征量纲会导致惩罚分配不公平。
标准化统一所有特征尺度，保证正则化对每个特征的约束公平有效，是正则化模型的必要前置步骤。

## 三、系数稳定性对比（50 次随机切分）
通过 50 次随机数据集划分，分别拟合 OLS 与 Ridge（α=10）：
- OLS：在 feat_1/2/3 共线性特征上系数波动极大，模型结果不稳定。
- Ridge：系数方差显著减小，输出结果稳定性大幅提升。
直观验证：正则化可有效缓解共线性带来的系数漂移问题。

## 四、GridSearchCV 超参数寻优
1. 搜索设置：
   - Ridge/Lasso：α ∈ np.logspace(-4, 3, 50)
   - ElasticNet：二维搜索 α + l1_ratio
2. 验证方式：5 折交叉验证
3. 结果：CV 误差呈典型 U 型曲线，最低点对应最优正则强度。

## 五、测试集模型性能
| 模型 | RMSE | MAE |
|------|------|-----|
| OLS | 1.2694 | 1.0117 |
| Ridge | 1.2645 | 1.0066 |
| Lasso | 1.2693 | 1.0205 |
| ElasticNet | 1.2647 | 1.0137 |

结论：Ridge 与 ElasticNet 精度略优于 OLS，Lasso 与 OLS 基本持平；正则化以微小精度损失换取了系数稳定性。

## 六、模型对共线性特征的处理行为
- Ridge：对 feat_1/2/3 均匀收缩，保留全部相关特征，不做剔除。
- Lasso：对共线性组执行稀疏选择，部分特征系数压缩至 0。
- ElasticNet：兼顾 L1 稀疏性与 L2 收缩性，保留特征组结构。
结果与课堂“模型性格”理论完全一致。

## 七、变量筛选结果对比
1. 自定义 CV 前向选择：`['feat_1', 'feat_4', 'feat_3', 'feat_5']`
2. Lasso 非零特征：`['feat_1', 'feat_4', 'feat_5', 'feat_6', 'feat_8']`
3. 共性：均识别出核心真实特征 feat_1、feat_4。
4. 差异：前向选择更聚焦强相关特征，Lasso 稀疏性更强，保留部分弱相关特征。
