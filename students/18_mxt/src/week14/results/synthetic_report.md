## Task A：自己生成数据，观察高维 / 共线性如何破坏 OLS

### A1. 高维且带有潜在低秩结构的数据

本次实验生成 `n = 180` 个样本、`p = 140` 个原始特征。特征不是彼此独立生成的，而是由 5 个潜在因子线性组合出来：

`X = F L + E`

其中 `F` 是 `180 x 5` 的潜在因子矩阵，`L` 是载荷矩阵，`E` 是较小的随机扰动。为了让共线性更明显，许多原始特征共享同一个主导潜在因子，只是在载荷和噪声上略有差异。

目标变量主要由潜在因子决定：

`y = 3.2 f1 - 2.4 f2 + 1.6 f3 + 1.1 f5 + noise`

也就是说，`y` 并不是由 140 个原始列各自独立决定的，而是由少数底层方向驱动。这正是 PCA/PCR 适合讨论的场景。

### A2. 数据保存与生成机制说明

合成数据保存在：

`src/week14/data/synthetic_highdim.csv`

这是一份“高维 + 信息冗余”的数据，原因有三点：

1. 特征数 `p = 140`，明显多于低层真实因子数 `5`。
2. 训练集样本数为 117，而最大实验维度是 140，因此出现 `p > n_train` 的高维场景。
3. 大量原始变量只是同一组潜在因子的不同噪声版本，变量之间存在强相关和信息重复。

图 `figures/matrix_structure_by_dimension.png` 展示矩阵结构随维度变化：

- 横轴：原始特征数量 `p`。
- 左纵轴：`rank(X_train)`，用柱状图表示。
- 右纵轴：condition number，使用对数刻度，用红色折线表示。

当 `p = 120` 或 `p = 140` 时，训练集设计矩阵的 rank 无法继续随特征数增长，condition number 变成无穷大。这说明 `X_train` 已经奇异或近似奇异，OLS 解会非常不稳定。

为什么“训练误差接近 0”反而危险？因为此时模型可能只是在用过多自由度记住训练样本，而不是学到稳定的预测规律。在线性代数上，`X^T X` 接近不可逆，很多不同系数组合都能解释训练集，但它们对新样本的预测可能差异很大。

### A4. 重复切分下的系数不稳定

本实验固定一份高维共线数据，进行 50 次不同随机 train/test split。每次都拟合 OLS，并收集三个关键变量 `x001`、`x002`、`x003` 的系数。

图 `figures/ols_coefficient_instability.png` 展示系数波动：

- 横轴：选定的原始变量。
- 纵轴：50 次随机切分下得到的 OLS 系数。
- 每个箱线图：同一个变量在不同训练集切分下的系数分布。

系数标准差如下：

| feature   |   coef_sd |
|:----------|----------:|
| x001      |    2.3961 |
| x002      |    1.6111 |
| x003      |    1.5482 |

观察结果是：测试误差会波动，系数波动更加明显。原因是 `x001`、`x002`、`x003` 都携带相近的潜在因子信息，OLS 在不同训练集中会把解释权分配给不同列。即使预测误差暂时不大，系数解释也可能完全不稳定。

课堂上说“系数不稳定本身就是风险”，原因在于：如果业务解释依赖“哪个变量重要、方向是什么、系数多大”，那么不稳定系数会导致结论随样本切分改变，无法作为可靠决策依据。

## Task B：PCA 与 PCR

### B1. PCA：前几个主成分是否解释大部分方差

图 `figures/pca_cumulative_variance.png` 展示 PCA 累计解释方差：

- 横轴：主成分个数。
- 纵轴：累计解释方差比例。
- 红色虚线：90% 累计解释方差参考线。

本次实验中，前 5 个主成分解释了 95.84% 的方差，达到 90% 累计解释方差需要 5 个主成分。这与数据生成机制一致：虽然原始空间有 140 维，但主要信息接近一个 5 维潜在子空间。

因此，“原始高维空间其实贴近更低维子空间”的含义是：许多原始列看起来数量很多，但它们的变化主要沿少数几个共同方向展开，剩余方向大多是噪声或细小扰动。

### B2. 自己组织 PCR 工作流

本次 PCR 没有把 sklearn pipeline 当黑箱直接交付，而是在 `utils.models.PCRRegressor` 中明确组织流程：

`标准化 X -> PCA -> 保留前 k 个主成分 -> 在 Z_k 上做 LinearRegression`

符号上可以写作：

`Z_k = X V_k`

然后在 `Z_k` 中拟合：

`y = Z_k gamma + error`

本实验比较 `k = 1` 到 `k = 20`，并记录 PCR 训练集 RMSE、PCR 测试集 RMSE、PCR CV RMSE。

### B3. CV 曲线解释

PCR CV RMSE 表示：在训练集内部做交叉验证时，模型对未参与拟合的验证折的平均预测误差。它比训练 RMSE 更适合选择 `k`，因为训练 RMSE 通常会随着模型复杂度增加而下降，但泛化误差不一定下降。

在本实验中，当 `k` 太小时，PCR 保留的信息不足，train/test/CV RMSE 都较高；当 `k` 接近真实潜在因子维度后，误差大幅下降；继续增加 `k` 后，训练误差可能略降，但 CV/test 改善有限，甚至可能变差。

OLS 在原始高维空间中可以取得很低训练误差，是因为自由度太高并且矩阵病态；这不代表 OLS 更好。相反，PCR 通过压缩到少数正交方向，牺牲少量无关细节，换来更稳定的泛化表现。

### B4. 公式定义与口头解释

OLS 估计式：

`beta_hat = (X^T X)^(-1) X^T y`

当 `X^T X` 不可逆或接近不可逆时，OLS 系数会不唯一或极不稳定。

第一主成分的方差最大化定义：

`v1 = argmax_{||v||=1} Var(Xv)`

它表示在所有单位方向中，找到让数据投影方差最大的方向。

PCR 的符号表达：

`Z_k = X V_k`

其中 `V_k` 是前 `k` 个主成分方向。之后不再直接用原始 `X` 回归，而是在压缩后的 `Z_k` 上回归。PCR 的核心思想不是“留下哪些原始变量”，而是“把冗余信息压缩到少数稳定方向”。
