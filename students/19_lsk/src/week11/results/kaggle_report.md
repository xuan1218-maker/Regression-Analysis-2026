# 🌍 Task B: Kaggle Real-World Regression Report

## 1. 数据集信息

- **数据集名称**: California Housing
- **来源**: sklearn.datasets.fetch_california_housing (源自 1990 年美国人口普查)
- **样本量**: 20,640
- **特征数**: 8 (数值型)
- **预测目标**: MedHouseVal — 加州街区组中位房价 (单位: $100,000)

### 业务含义:
每条样本代表加州一个街区组 (block group) 的房屋信息。
目标是通过收入、房龄、地理位置等特征预测该区域的中位房价。

### 为什么选择这个数据集:
1. 它是真实人口普查数据, 不是"演示型"教学数据, 存在真实的数据噪声和分布特征;
2. 包含多种数值型特征, 且经纬度可以构造类别变量;
3. 样本量足够大 (20,640), 适合交叉验证;
4. 房价预测是一个经典的回归问题, 业务上易于理解。

### 数据加载方式
由于在线 `fetch_california_housing` 返回 HTTP 403, 使用本地 `housing.csv` 文件回退加载,
并将列名映射为 sklearn 兼容格式:
- `median_income` → `MedInc`
- `housing_median_age` → `HouseAge`
- `total_rooms / households` → `AveRooms`
- `total_bedrooms / households` → `AveBedrms`
- `population` → `Population`
- `population / households` → `AveOccup`
- `latitude` → `Latitude`
- `longitude` → `Longitude`
- `median_house_value / 100000` → `MedHouseVal` (转换为 $100k 单位)

## 2. 数据清洗与预处理

- **类别变量构造**: 基于 `Latitude` 将加州划分为三个区域:
  - `SouthCA` (Lat < 35.0)
  - `CentralCA` (35.0 ≤ Lat < 37.0)
  - `NorthCA` (Lat ≥ 37.0)
- **缺失值**: 发现 `AveBedrms` 有 207 个缺失值 (由 `total_bedrooms` 缺失导致), 用中位数填充
- **异常值**: 对所有连续数值特征 (除目标变量和经纬度) 使用 1%-99% Winsorization
- **编码**: `region_ca` 使用 One-Hot 编码 (drop first)
- **标准化**: 使用自定义 `CustomStandardScaler`

## 7. 推测分析

### 7.1 最稳定的变量
`MedInc` (中位收入) 通常与房价关系最稳定且正向最强,
这与经济学直觉 (收入越高, 房价越高) 一致。

### 7.2 不稳定的变量
`AveRooms` 和 `AveBedrms` 可能由于尺度或相关性问题而不稳定。
`Population` 和 `AveOccup` 的效应方向在实际中可能依赖于具体区域。

### 7.3 共线性问题
房间数和卧室数 (`AveRooms` ↔ `AveBedrms`) 往往相关, 可能导致 VIF 偏高。
实际 VIF 分别为 3.57 和 1.87, 属于可接受范围。
更严重的共线性来自经纬度与区域变量的关系。

### 7.4 业务解释
模型的平均 RMSE 约为 $0.6561 (单位: $100k),
即预测误差约 $65,614 美元。
在业务上这意味着: 对一栋中位房价的房屋, 模型预测误差在可接受范围内。

### 7.5 上线风险
1. **地理漂移**: 数据集是 1990 年的, 当前加州房价结构已发生显著变化;
2. **分布外推**: 对极端收入区域 (极富/极贫) 预测可能不准;
3. **遗漏变量**: 利率、学区质量等重要因素未在数据中;
4. **共线性**: 经纬度与区域变量的共线性可能导致政策解读出错。
