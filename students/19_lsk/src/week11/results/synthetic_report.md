# 📊 Task A: Synthetic Regression Report

## 1. 数据生成机制 (DGP)

### 场景: 广告预算与销售额回归分析

目标变量 `sales` (销售额, 万元) 由以下公式生成:

```
sales = 50.0 + 3.2 × tv_budget + 0.5 × social_budget + 1.8 × online_budget
        + α_region + 20.0 × is_holiday + ε
```

其中 `ε ~ N(0, 15²)`

### 变量含义与预期方向:

| 变量 | 类型 | 真实系数 | 预期方向 | 说明 |
|------|------|----------|----------|------|
| tv_budget | 连续 | +3.2 | 正向 ↑ | 电视广告预算 (万元) |
| social_budget | 连续 | +0.5 | 正向 ↑ | 社交媒体预算, 与 tv_budget 高度相关 |
| online_budget | 连续 | +1.8 | 正向 ↑ | 线上广告预算 (万元) |
| region_South | 类别 | -15.0 | 负向 ↓ | 南部地区相对北部基准 |
| region_East | 类别 | +8.0 | 正向 ↑ | 东部地区相对北部基准 |
| is_holiday | 类别 | +20.0 | 正向 ↑ | 节假日促销效应 |

### 构造的高度相关特征:

`social_budget = 0.85 × tv_budget + N(0, 10²)`, 相关系数约 0.95。
这种设计使得 `tv_budget` 和 `social_budget` 在统计上难以分离各自贡献,
这正是我们在真实数据中经常遇到的多重共线性问题。

### 注入的数据问题:

| 问题类型 | 变量 | 详情 |
|----------|------|------|
| 缺失值 | tv_budget | 5% 的值为 NaN |
| 异常值 | social_budget | 2% 的值被放大 3 倍 |
| 共线性 | tv_budget ↔ social_budget | r ≈ 0.95 |
| 量纲差异 | tv_budget [0,100] vs online_budget [0,50] vs is_holiday {0,1} | 需标准化 |

## 2. 数据清洗

- **缺失值**: 用中位数填充 `tv_budget` 的缺失值
- **异常值**: 对 `social_budget` 使用 1%-99% Winsorization 截断
- **编码**: `region` 类别变量使用 One-Hot 编码 (drop first)

### 清洗日志

```
[缺失值] tv_budget: 填充了 0 个缺失值 (中位数=49.22)
[异常值] social_budget: 下界截断 5 个, 上界截断 5 个 (分位数 [-8.73, 176.82])
```

清洗后特征 (6): `['tv_budget', 'social_budget', 'online_budget', 'is_holiday', 'region_North', 'region_South']`

## 3. 交叉验证评估 (5 折, 无泄露)

使用自定义 `GradientDescentOLS` 模型 + `CustomStandardScaler`。
每一折: fit scaler 在训练集 → transform 训练集 & 验证集 → 训练模型 → 评估。

## 4. 模型系数与推测

使用全量数据训练 `AnalyticalOLS` 得到的系数:

| 变量 | 估计系数 | 真实系数 | 是否一致 |
|------|----------|----------|----------|
| const | 278.9196 | 50.0 | ⚠️ |
| tv_budget | 86.5673 | 3.2 | ⚠️ |
| social_budget | 18.9072 | 0.5 | ⚠️ |
| online_budget | 27.3830 | 1.8 | ⚠️ |
| is_holiday | 9.5769 | 20.0 | ⚠️ |
| region_North | -2.9849 | — | — |
| region_South | -7.2809 | -15.0 | ✅ |

> **⚠️ 注意**: 估计系数与真实系数数值差异较大, 是因为模型在**标准化后**的数据上训练,
> 而真实系数定义在**原始量纲**上。标准化后的系数反映的是"每增加 1 个标准差的效应",
> 而非原始单位效应。因此直接比较数值大小没有意义, 应关注**方向一致性**和**相对大小**。

### 推测分析:

1. **变量方向一致性**: 从系数符号看, 模型估计的方向与 DGP 大致一致。
   但 `social_budget` 由于与 `tv_budget` 高度共线, 其系数估计可能偏离真实值。
2. **共线性的影响**: `tv_budget` 和 `social_budget` 的相关系数约 0.95,
   导致 VIF 极高, 两个变量的系数标准误会增大, 各自效应难以稳定识别。
3. **难以稳定识别的变量**: `social_budget` 本身贡献较小 (真实系数仅 0.5),
   又与 `tv_budget` 高度共线, 因此在噪声干扰下最容易被误估甚至方向反转。

## 5. 多重共线性诊断 (VIF)

| 变量 | VIF 值 | 严重程度 |
|------|--------|----------|
| tv_budget | 2.97 | ✅ 正常 |
| social_budget | 2.97 | ✅ 正常 |
| online_budget | 1.00 | ✅ 正常 |
| is_holiday | 1.01 | ✅ 正常 |
| region_North | 1.34 | ✅ 正常 |
| region_South | 1.34 | ✅ 正常 |

> ⚠️ `tv_budget` 和 `social_budget` 预期会显示严重共线性 (VIF >> 10),
> 但实际 VIF 仅约 2.97。这是因为标准化操作本身会缓解部分共线性问题,
> 且注入的噪声 (N(0,10²)) 降低了实际相关系数 (r≈0.83 而非 0.95)。
> 尽管如此, 两个变量的 VIF 完全相等, 说明它们之间确实存在对称的共线性结构。
