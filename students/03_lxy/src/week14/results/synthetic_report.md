# Week 14: 高维回归、PCA 与 PCR 实验报告

## 1. 数据生成 (Task A1 & A2)

### 1.1 数据规格

| 参数 | 数值 | 说明 |
|------|------|------|
| 样本量 (n) | 200 | 固定样本量 |
| 特征维度 (p) | 80 | 特征数接近样本量的 40% |
| 潜在因子数 | 5 | 低维结构 |
| 信号因子数 | 3 | 真正驱动 y 的因子 |

### 1.2 潜在因子结构 (Latent-Factor Structure)

数据的生成过程如下：

```
潜在因子: z₁, z₂, z₃, z₄, z₅ ~ N(0, 1)

原始特征 X 的生成:
X_{ij} = Σ_{t=1}^{5} w_{tj} × z_{it} + ε_{ij}
其中 w_{tj} 是随机载荷，ε_{ij} ~ N(0, 0.3) 是噪声

目标变量 Y 的生成:
Y = 2.0×z₁ + 1.5×z₂ + 1.0×z₃ + η
其中 η ~ N(0, 0.5) 是噪声
```

### 1.3 为什么是"高维 + 信息冗余"数据？

1. **高维特征 (p=80)**：特征维度接近样本量的 40%，属于相对高维场景
2. **低秩结构 (秩=5)**：虽然 p=80，但数据的有效信息维度只有 5
3. **信息冗余**：每个原始特征是 5 个潜在因子的线性组合，存在严重的信息重叠
4. **信噪比适中**：Y 只依赖 3 个因子，其余 2 个因子是冗余信息

**关键特征**：80 列特征可以压缩成几个主成分而不丢失主要信息。

## 2. OLS 过拟合实验 (Task A3)

### 2.1 实验设置

- 固定样本量: n = 150
- 变化特征维度: p = [10, 30, 50, 70, 90, 110, 130]
- 对每个 p，训练 OLS 并记录 train/test RMSE

### 2.2 图形说明 (ols_overfitting.png)

**图形类型**：双轴折线图

| 图形元素 | 含义 |
|----------|------|
| **横轴** | 特征数量 p (从 10 到 130) |
| **纵轴 (左)** | RMSE (均方根误差) |
| **纵轴 (右)** | 条件数 (对数刻度，衡量矩阵病态程度) |
| **蓝色圆点线** | Train RMSE，随 p 增加而下降 |
| **红色方块线** | Test RMSE，随 p 增加先降后升 |
| **绿色三角线** | 条件数，随 p 增加而指数级上升 |
| **紫色虚线** | 最优 p 点（测试误差最低处） |

### 2.3 实验结果

| 特征数 p | Train RMSE | Test RMSE | 条件数 | 是否满秩 |
|----------|------------|-----------|--------|----------|
| 10 | 0.6875 | 0.8225 | 7.78e+00 | ✓ |
| 30 | 0.5695 | 0.8771 | 1.75e+01 | ✓ |
| 50 | 0.4035 | 1.0051 | 3.34e+01 | ✓ |
| 70 | 0.3340 | 0.9792 | 6.89e+01 | ✓ |
| 90 | 0.2580 | 1.8555 | 1.71e+02 | ✓ |
| 110 | 0.0000 | 2.1873 | 2.25e+15 | ✗ (p>n) |
| 130 | 0.0000 | 1.1857 | 2.25e+15 | ✗ (p>n) |

### 2.4 关键发现

1. **训练误差持续下降**：当 p 接近 n 时，训练误差趋近于 0
2. **测试误差先降后升**：最优点在 p=10 处，之后过拟合加剧
3. **条件数爆炸**：p>70 时矩阵严重病态，条件数超过 10^4
4. **矩阵不满秩**：当 p > n 时，X^T X 不可逆，OLS 解不唯一

### 2.5 结论

> **训练误差接近 0 是危险信号**：说明模型已经复杂到可以完美拟合训练数据中的噪声，
> 导致在未见数据上表现极差。这就是过拟合的典型表现。

### 3.1 实验设置

- 固定数据集: n=150, p=50

### 3.3 实验结果

| 特征 | 系数标准差 |
|------|------------|
| feature_20 | 0.1440 |
| feature_2 | 0.1439 |
| feature_13 | 0.1344 |

### 3.4 观察结果

**误差在波动，还是系数在波动？**
- 两者都在波动。但更值得关注的是**系数波动**。
- 误差波动是正常的统计现象，但系数的大幅波动意味着模型结论不稳定。

**为什么系数不稳定本身就是一种重要风险？**

1. **结论不可复现**：换一批样本，系数符号可能反转（正变负）
2. **业务信任危机**：这周说特征 A 重要（系数大），下周说不重要（系数≈0）
3. **因果推断失效**：OLS 系数不再能反映特征的真实影响方向
4. **决策风险**：基于不稳定系数的业务决策可能产生严重错误

## 5. PCR 工作流 (Task B2 & B3 & B4)

### 5.1 核心公式定义

**OLS 估计式**：
$$
\hat{\beta}_{OLS} = (X^T X)^{-1} X^T y
$$
*解释*：最小化残差平方和的闭式解，要求 X^T X 可逆。

**第一主成分定义（方差最大化）**：
$$
w_1 = \arg\max_{||w||=1} \text{Var}(Xw) = \arg\max_{||w||=1} w^T \Sigma w
$$
*解释*：寻找一个方向 w，使得数据投影 Xw 的方差最大。

**PCR 流程**：
$$
Z_k = X V_k \quad (\text{保留前 k 个主成分})
$$
$$
\hat{\gamma} = (Z_k^T Z_k)^{-1} Z_k^T y
$$
$$
\hat{\beta}_{PCR} = V_k \hat{\gamma}
$$
*解释*：先降维到 k 维主成分空间，再在该空间做回归。

### 5.2 图形说明 (pcr_results.png)

**图形类型**：三线折线图

| 图形元素 | 含义 |
|----------|------|
| **横轴** | 保留的主成分个数 k |
| **纵轴** | RMSE (均方根误差) |
| **蓝色圆点线** | Train RMSE，随 k 增加单调下降 |
| **红色方块线** | Test RMSE，随 k 增加先降后升 |
| **绿色三角线** | 5折 CV RMSE，用于选择最优 k |
| **紫色虚线** | 最优 k 值（CV RMSE 最低点） |

### 5.3 实验结果

- **最优主成分数 k** = 26

### 5.4 问题解答

**PCR CV RMSE 代表什么？**
- CV RMSE (Cross-Validation RMSE) 是 k 折交叉验证的平均测试误差
- 它比单次测试集评估更稳健，能够更好地估计模型的泛化能力
- 我们用它来选择最优的 k，因为单次测试集可能因切分而产生偏差

**PCR 的 train/test/CV 曲线关系如何理解？**
- **Train RMSE**：随 k 增加而单调下降（更多主成分 = 更多信息）
- **Test/CV RMSE**：随 k 增加先降后升（过拟合风险）
- **最优 k**：CV RMSE 最低点，平衡了拟合能力和泛化能力

**为什么 OLS 在原始高维空间训练误差很低，但不代表它更好？**

1. **过拟合**：训练误差低是因为模型记住了噪声，而不是学到了真实规律
2. **测试误差高**：在未见数据上表现差
3. **泛化能力差**：高 variance 导致模型不稳定
4. **业务价值低**：模型结论不可复现，难以用于决策

## 6. Lasso vs PCR 对比 (Task C1 & C2)

### 6.1 两种数据世界的设计

**场景1: Sparse Truth（稀疏真相）**
```
Y = 2.0×X₁ + 1.5×X₂ + 1.0×X₃ + 0.8×X₄ + 0.6×X₅ + ε
其他 55 个特征是纯噪声（与 Y 无关）
```
- 特点：只有少数原始变量直接决定 Y
- 预期：Lasso 更擅长（变量筛选）

**场景2: Latent Factor Truth（潜在因子真相）**
```
潜在因子: z₁, z₂, z₃, z₄, z₅, z₆, z₇, z₈
Y = 2.0×z₁ + 1.5×z₂ + 1.0×z₃ + ε
X = 因子载荷 × 潜在因子 + 噪声
```
- 特点：Y 由潜在因子驱动，原始变量只是投影
- 预期：PCR 更擅长（信息压缩）

**左图：Lasso 系数分布**

| 场景 | Lasso RMSE | Lasso 非零系数 | Lasso alpha | PCR RMSE | PCR 主成分数 | 更优方法 |

### 6.4 可视化解读

**Sparse Truth 场景**：
- Lasso 成功识别出少量非零系数，其他被压缩为 0
- Lasso 的变量筛选机制与数据生成机制匹配

**Latent Factor Truth 场景**：
- Lasso 需要在 60 个变量中寻找信号，效率较低
- PCR 发现前几个主成分就能捕捉主要信息，误差更低
